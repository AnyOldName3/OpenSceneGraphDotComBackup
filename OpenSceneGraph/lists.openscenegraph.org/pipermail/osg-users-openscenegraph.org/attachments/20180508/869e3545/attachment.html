<p class="MsoNormal" style="margin-bottom:12.0pt">Hi Marlin,<o:p></o:p></p>
</div>
<p class="MsoNormal" style="margin-bottom:12.0pt">The Transform class and it's subclasses are designed to be used as part of scene graph, this allow Transform subclasses that are absolute (ignore transforms+camera matrices above them) or relative (multiple
 the inherited modelview matrix) to enable this in a generic way the Transform class has two helper methods Transform::computeLocalToWorld(osg::Matrix& matrix, NodeVistor*) and computeWorldToLocal(osg::Matrix& matrix, NodeVisior*). Only some Transform implementations
 need to check the NodeVisitor for extra data, most will just ignore that parameter and just do the matrix maths required.<o:p></o:p></p>
</div>
<p class="MsoNormal" style="margin-bottom:12.0pt">The computeLocalToWorld() method returns a bool on success, and you pass in the matrix you want to apply,  You're code segment appears to confuse things in this regard. 
<br>
<br>
<o:p></o:p></p>
</div>
<p class="MsoNormal" style="margin-bottom:12.0pt">If you want to transform of any node in the scene graph you can do Node::getWorldMatrices(), this finds all the parental NodePaths of that node and then returns a vector containing a matrix for each NodePath
 found.  <o:p></o:p></p>
</div>
