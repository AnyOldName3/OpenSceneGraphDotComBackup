<div dir="ltr">To clarify, UWP is a specialized sub/superset of the Windows platform APIs, which targets modern Windows devices like XBox, Windows Mobile and HoloLens. It adds a number of modern APIs but also excludes a number of POSIX-like APIs as well as precluding OpenGL itself (!). The way to work around this for OSG, of course is to utilize ANGLE ( <a href="https://github.com/google/angle">https://github.com/google/angle</a> ) and OSG's OpenGL ES target to run on DirectX.<div><br></div><div>It's a big hairball of work-arounds, but it would enable OSG to run on desirable targets like HoloLens which is compelling enough to maybe make it worthwhile.</div><div><br></div><div><br></div>​I know of at least a few developers trying to achieve this so there could be a small amount of community testing.<div><br></div><div><br></div></div>
